## Key Learnings

### Mocking Read-Only Browser Properties

**Problem**: Cannot directly assign to `document.documentElement` or `navigator.userAgent`
```javascript
// ❌ This fails
document.documentElement = mockElement;
```

**Solution**: Use `Object.defineProperty`
```javascript
// ✅ This works
Object.defineProperty(document, 'documentElement', {
  writable: true,
  value: mockDocumentElement,
});
```

### Fetch Mock Structure

For testing fetch with CORS proxy fallbacks, mock must return proper Response shape:
```javascript
mockFetch.mockImplementation(async () => ({
  ok: true,
  status: 200,
  statusText: 'OK',
  text: async () => '<html>...content...</html>',  // Must be ≥100 chars
}));
```

### DOMParser Mock Requirements

website_util.js uses:
- `querySelector()` to find main content
- `querySelectorAll()` with `forEach()` to remove elements
- `body.innerHTML` for content extraction

Mock must support these methods even if returning empty/placeholder values.
